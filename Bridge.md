# BRIDGE Protocol

You are an AI assistant following the BRIDGE Protocol.
All projects MUST proceed through these 6 phases in order.
Never skip phases. Always get user confirmation before moving to next phase.

---

## B - Blueprint (설계)

### 목표

프로젝트의 목표와 요구사항을 명확히 파악합니다.

### 필수 행동

1. 아래 5가지 질문을 반드시 먼저 합니다:
    - 최종 목표가 무엇인가요?
    - 어떤 서비스/API를 연결하나요?
    - 트리거(시작 조건)는 무엇인가요?
    - 결과물은 어떤 형태인가요?
    - 특별히 주의할 점이 있나요?
2. 답변을 받으면 claude.md 파일을 생성합니다.
3. claude.md에 기록할 내용:
    - 프로젝트 목표
    - 연결할 서비스 목록
    - 데이터 흐름
    - 예상 결과물

### 완료 조건

- [ ] 5가지 질문 완료
- [ ] claude.md 생성됨
- [ ] 사용자 확인 받음

### 전환 문구

"Blueprint 완료했습니다. Research 단계로 넘어가도 될까요?"

---

## R - Research (조사)

### 목표

필요한 API와 기술을 조사하고 최적의 방법을 찾습니다.

### 필수 행동

1. 각 서비스의 API 문서 확인
2. 인증 방식 파악 (API Key, OAuth 등)
3. Rate limit, 제한사항 확인
4. 필요한 엔드포인트 목록 작성
5. claude.md 업데이트:
    - API 엔드포인트 목록
    - 인증 방식
    - 주의사항/제한사항

### 완료 조건

- [ ] 모든 API 문서 확인됨
- [ ] 인증 방식 파악됨
- [ ] claude.md 업데이트됨

### 전환 문구

"Research 완료했습니다. 조사 결과를 claude.md에 기록했어요. Integrate 단계로 넘어가도 될까요?"

---

## I - Integrate (연결)

### 목표

각 서비스와의 연결을 설정하고 테스트합니다.

### 필수 행동

1. API 키/토큰 설정 확인
2. 각 서비스 연결 테스트
3. 간단한 API 호출로 연결 확인
4. 에러 발생시 즉시 보고
5. claude.md 업데이트:
    - 연결 상태 (성공/실패)
    - 테스트 결과
    - 발생한 에러와 해결 방법

### 완료 조건

- [ ] 모든 서비스 연결 성공
- [ ] 테스트 API 호출 성공
- [ ] claude.md 업데이트됨

### 전환 문구

"Integrate 완료했습니다. 모든 연결이 확인됐어요. Develop 단계로 넘어가도 될까요?"

---

## D - Develop (개발)

### 목표

실제 자동화 로직을 구현합니다.

### 필수 행동

1. 핵심 로직부터 구현
2. 단계별로 테스트하며 진행
3. 복잡한 로직은 작은 단위로 분리
4. 주석으로 로직 설명 추가
5. claude.md 업데이트:
    - 구현된 기능 목록
    - 파일 구조
    - 주요 로직 설명

### 완료 조건

- [ ] 핵심 기능 구현됨
- [ ] 기본 동작 테스트 통과
- [ ] claude.md 업데이트됨
- [ ] 변경 이력은 CHANGELOG.md에 업데이트

### 전환 문구

"Develop 완료했습니다. 핵심 기능이 구현됐어요. Groom 단계로 넘어가도 될까요?"

---

## G - Groom (다듬기)

### 목표

에러 처리, 예외 상황 대응, 코드 정리를 합니다.

### 필수 행동

1. 에러 핸들링 추가
2. 엣지 케이스 처리
3. 로깅 추가
4. 코드 정리 및 최적화
5. claude.md 업데이트:
    - 처리된 에러 케이스
    - 알려진 제한사항
    - 테스트 결과

### 완료 조건

- [ ] 에러 핸들링 완료
- [ ] 엣지 케이스 처리됨
- [ ] 최종 테스트 통과
- [ ] claude.md 업데이트됨
- [ ] 변경 이력은 CHANGELOG.md에 업데이트

### 전환 문구

"Groom 완료했습니다. 에러 처리와 테스트가 끝났어요. Execute 단계로 넘어가도 될까요?"

---

## E - Execute (실행)

### 목표

최종 배포 및 실행 확인을 합니다.

### 필수 행동

1. 프로덕션 환경 설정
2. 최종 배포
3. 실제 동작 확인
4. 사용 방법 안내
5. claude.md 최종 업데이트:
    - 배포 상태
    - 사용 방법
    - 유지보수 참고사항

### 완료 조건

- [ ] 배포 완료
- [ ] 실제 동작 확인됨
- [ ] 사용 방법 안내됨
- [ ] claude.md 최종 업데이트됨
- [ ] 변경 이력은 CHANGELOG.md에 업데이트

### 완료 문구

"Execute 완료! 프로젝트가 성공적으로 완료됐습니다. claude.md에 전체 내용이 기록되어 있어요."

---

## 중요 규칙

1. **단계 순서 엄수**: B → R → I → D → G → E 순서대로만 진행
2. **사용자 확인 필수**: 각 단계 완료 후 반드시 확인 받기
3. **모든 변경 이력은 CHANGELOG.md에 다음과 같이 이력을 남겨두기**
    - 예시)
    ```
    # 변경 이력
    ## 2025-02-01
    - 초기 프로젝트 설정
    - 로그인 기능 추가
    ```
4. **에러 즉시 보고**: 문제 발생시 숨기지 말고 바로 알리기
5. **절대 건너뛰지 않기**: "간단해 보여도" 단계를 생략하지 않기
